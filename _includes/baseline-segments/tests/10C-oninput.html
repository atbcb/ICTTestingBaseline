10.C Test Procedure for On Input

Baseline Test ID: 10.C-OnInput

Identify Content

All active form components.

Test Instructions

  1. Enter data in all form fields, and exit (tab out of) the field.
  2. Check that navigating away from a component and/or changing component values/selections (e.g., entering data in a text field, changing a radio button selection) does NOT initiate a change of context unless the user has been advised of the behavior before using the component. [SC 3.2.2]

Test Results

If any of the above checks fail, then Baseline Test 10.C-OnInput fails.
