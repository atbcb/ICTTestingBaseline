Test method rationale

Headers and associations are critical for conveying tabular relationships to assistive technologies; these checks ensure semantic structure is preserved.

Method Rationale

For assistive technology (AT) users, data tables must explicitly associate table data with table row and column headers via programmatic markup. Table markup also facilitates navigation for AT users by providing programmatic landmarks via column and row headers. When <table> elements are used for layout purposes, data table structure elements are not permitted, such as <th>, <caption>, or <summary> (HTML4).
